为什么要在本地部署DeepSeek R1

DeepSeek R1是深度求索推出的开源推理模型。本地部署的好处：数据隐私完全可控、无API费用、无网络延迟。缺点是需要一块不错的显卡。

硬件要求速查表

模型大小最低显存推荐显存系统内存
7B6 GB8 GB16 GB
14B12 GB16 GB32 GB
32B24 GB24 GB+64 GB
70B48 GB80 GB128 GB
671B—多卡集群512 GB+

以上数据综合自开发者社区实测反馈（Facebook Developer Kaki等）。7B模型在RTX 3060上运行流畅，14B在RTX 3060上可运行但token速度下降。

方案一：8GB显存——用Ollama跑7B

使用Ollama是最简单的方案（支持Windows/Mac/Linux）：

# 下载：https://ollama.com
ollama run deepseek-r1:7b

7B量化版在8GB显卡上实际占用约5.5GB显存，生成速度约15-25 token/秒，日常对话完全够用。

方案二：12GB显存——跑14B模型

RTX 3060 12GB性价比极高（二手约1500元）：

ollama run deepseek-r1:14b

14B实际占用约10GB显存，速度约8-15 token/秒。

方案三：24GB显存——跑32B模型

RTX 3090或4090是最佳选择：

ollama run deepseek-r1:32b

32B模型推理能力显著强于14B，在数学证明、代码生成上接近70B。速度约20-30 token/秒。

方案四：48GB显存——跑70B

需要A6000/L40S等专业卡或双卡。社区反馈70B在48GB显存上运行仍然偏慢。如果追求推理质量，优先考虑云端API。

常见问题

CPU-only可以跑吗？可以，但7B约1-2 token/秒，仅适合测试。

Mac能跑吗？可以。M2 Ultra 192GB可跑70B，M3 Max 128GB可跑32B。Ollama原生支持Apple Silicon。
